# Security Policy

Please report security issues privately to security@borgels.com.

Do not open public GitHub issues for suspected vulnerabilities, leaked
credentials, or sensitive operational details.

When reporting, include:

- Affected repository and version or commit.
- Reproduction steps or a concise proof of concept.
- Expected impact.
- Any relevant logs with secrets redacted.

We aim to acknowledge reports within a reasonable timeframe and coordinate fixes
before public disclosure where appropriate.
